# 塔罗插件工作流（当前版本）

本文档描述插件当前已经实现并上线的占卜流程（v0.2.4）。

## 1. 当前能力概览

1. 支持用户使用 /tarot 问题 发起多牌占卜。
2. 支持用户只输入 /tarot 时返回提问引导。
3. 支持单张占卜（塔罗牌 问题）与多牌占卜并行。
4. 支持单牌解读与整体 AI 解读。
5. 支持结果记录与历史查询。
6. 支持群聊合并转发开关。

## 2. 用户交互流程

1. 用户发起 /tarot 问题 或 塔罗牌 问题。
2. 插件返回准备提示：静心、净化、洗牌文案。
3. 插件根据问题匹配牌阵（单张流程固定单张牌阵）。
4. 插件构建抽牌编号池并返回可选编号列表（默认展示该主题全部可抽牌）。
5. 用户发送 抽牌 编号1 编号2 ...。
6. 插件校验编号后翻牌，逐张返回：位置 + 正逆位 + 图片 + 单牌解读。
7. 插件返回整体 AI 解读。
8. 插件将结果写入记录文件。

## 3. 命令说明

1. /tarot 问题
- 进入多牌占卜流程。

2. /tarot
- 不带问题时，返回提问引导。

3. 塔罗牌 问题
- 进入单张占卜流程。

4. 抽牌 编号1 编号2 ...
- 根据占卜提示完成抽牌。
- 编号数量必须与牌阵要求一致。

5. 占卜记录 数量
- 查看当前会话用户最近记录。
- 默认 3 条，最多 10 条。

6. 开启群聊转发 / 关闭群聊转发
- 切换群聊下的消息发送方式。

## 4. 阶段化流程（已实现）

| 阶段 | 名称 | 当前实现 |
|---|---|---|
| S0 | 会话启动 | 接收问题并创建待抽牌会话 |
| S1 | 准备提示 | 发送静心与洗牌文案 |
| S2 | 问题标准化 | 清理输入，空问题使用默认问句 |
| S3 | 牌阵选择 | 多牌走匹配逻辑，单张固定 1 张 |
| S4 | 编号抽牌 | 返回编号池，等待用户输入编号 |
| S5 | 翻牌解读 | 返回牌面图、正逆位、单牌解读 |
| S6 | 整体解读 | 输出整体 AI 分析 |
| S7 | 结果记录 | 写入本地记录文件并支持查询 |

## 5. 关键实现位置

1. 会话创建与牌阵准备：main.py 中 _create_pending_session。
2. 多牌入口：main.py 中 tarot_handler -> divine。
3. 单张入口：main.py 中 onetime_divine_handler -> onetime_divine。
4. 编号抽牌执行：main.py 中 draw_by_numbers。
5. 翻牌与单牌/整体解读：main.py 中 _reveal_cards。
6. 记录查询：main.py 中 show_records。

## 6. 运行参数

1. pending_expire_seconds
- 待抽牌会话超时时间（秒）。

2. draw_pool_factor
- 编号抽牌池倍率。
- <=0 时展示全部可抽牌；>0 时按倍率压缩抽牌池。

3. enable_record
- 是否启用记录写入与查询。

## 7. 异常处理

1. 未发起占卜直接抽牌：提示先发起占卜。
2. 会话超时：提示重新发起。
3. 编号数量不匹配、重复或越界：提示重新输入。
4. 图片缺失或 AI 失败：返回错误提示并中断当前步骤。
