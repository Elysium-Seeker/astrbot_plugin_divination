# 塔罗插件工作流（当前版本）

本文档描述插件当前已经实现并上线的占卜流程（v0.2.9）。

## 1. 核心状态机

1. Idle：无待抽牌会话。
2. PendingDraw：已创建牌阵与候选池，等待用户输入编号。
3. Revealing：执行翻牌与单牌解读。
4. Completed：输出整体解读并落盘记录，回到 Idle。

## 2. 入口优先级（非常重要）

1. 直回编号入口
- 条件：消息为纯数字且存在待抽牌会话。
- 行为：直接进入抽牌流程。

2. /tarot 编号兜底入口
- 条件：用户显式发送 /tarot 1 2 3。
- 行为：若有待抽牌会话则直接抽牌；无会话则提示先发起占卜。

3. /tarot 问题入口
- 条件：/tarot 后跟自然语言问题。
- 行为：创建待抽牌会话并返回候选编号池。

4. 单张入口
- 条件：塔罗牌 问题。
- 行为：创建单张待抽牌会话并返回候选编号池。

## 3. 用户交互流程

1. 用户发起 /tarot 问题 或 塔罗牌 问题。
2. 插件返回准备提示（静心、净化、洗牌文案）。
3. 插件匹配牌阵并生成候选池（默认全牌池）。
4. 用户通过以下任一方式输入编号：
- 直接回复编号（推荐）。
- 抽牌 编号1 编号2 ...。
- /tarot 编号1 编号2 ...（分流兜底）。
5. 插件校验编号后翻牌，逐张返回：位置 + 正逆位 + 图片 + 单牌解读。
6. 插件返回整体 AI 解读。
7. 插件写入占卜记录文件。

## 4. 命令说明

1. /tarot 问题
- 启动多牌占卜。

2. /tarot
- 不带问题时返回提问引导。

3. /tarot 编号1 编号2 ...
- 待抽牌会话下作为命令兜底抽牌。

4. 塔罗牌 问题
- 启动单张占卜。

5. 抽牌 编号1 编号2 ...
- 传统抽牌命令路径。

6. 占卜记录 数量
- 查看当前会话用户的最近记录（默认 3，最多 10）。

7. 开启群聊转发 / 关闭群聊转发
- 切换群聊发送方式。

## 5. 关键实现位置

1. 会话创建：main.py 中 _create_pending_session。
2. 主入口：main.py 中 tarot_handler。
3. 自动编号入口：main.py 中 draw_by_reply_handler。
4. 抽牌执行：main.py 中 draw_by_numbers。
5. 翻牌与解读：main.py 中 _reveal_cards。
6. 记录查询：main.py 中 show_records。

## 6. 运行参数

1. pending_expire_seconds
- 待抽牌会话超时时间（秒）。

2. full_draw_pool
- 是否强制全牌池（默认 true）。
- 开启时忽略 draw_pool_factor。

3. draw_pool_factor
- 候选池倍率，仅在 full_draw_pool=false 时生效。

4. enable_record
- 是否启用记录写入与查询。

## 7. 异常处理

1. 无会话抽牌：提示先发起占卜。
2. 会话超时：提示重新发起。
3. 编号数量不匹配、重复、越界：提示重新输入。
4. 图片缺失或 AI 失败：返回错误提示并中断当前步骤。
